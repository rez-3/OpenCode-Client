# 项目架构深度分析

## 一、项目定位

一个 **Wails v2 桌面应用**，为 [OpenCode](https://github.com/anomalyco/opencode) AI 编程助手提供**可视化管理中心**。相当于 OpenCode 的"控制面板"——不替代 OpenCode 本身，而是提供一个图形界面来管理其配置、会话、技能和 web 服务。

---

## 二、技术栈总览

| 维度 | 技术选择 |
|------|---------|
| **桌面框架** | Wails v2.12（Go 后端 + WebView2 前端） |
| **后端语言** | Go 1.25 |
| **前端** | 原生 HTML/CSS/JS（**无 React/Vue 等框架**） |
| **Markdown 渲染** | marked.js |
| **代码规模** | Go ~2700行 / JS ~3800行 / CSS ~2950行 / HTML ~280行 |
| **构建产物** | `oc-manager.exe`（~12MB，Go embed 前端资源） |
| **测试** | 3 个测试文件，Go 原生 `testing` 包 |

```

 ├── App 结构体 ──── 60+ 导出方法，所有前端调用的入口
 │    ├── skill.Manager      → 技能符号链接管理

 │
 ├── model/    共享类型定义（前后端数据结构契约）
 ├── config/   配置文件读写（JSONC 注释保留）
 ├── service/  OpenCode serve 进程管理 + HTTP 代理
 └── skill/    技能目录扫描 + 符号链接 toggle
```

### Wails 绑定模式

```go
// main.go - 绑定入口
Bind: []interface{}{app}  // App 结构体的所有公开方法自动暴露给前端

// 前端调用示例
window.go.main.App.StartOpenCodeWeb(4096, "127.0.0.1", proxyConfig)
window.go.main.App.GetSkills()
window.go.main.App.OpenCodeAPI("GET", "/session", "")
```

## 五、前端架构概览

> 详情见 [前端模块](modules/前端模块.md)

### 技术特点
- **零框架依赖**：纯原生 JavaScript，无 React/Vue/Angular
- **模块化加载**：12 个 JS 文件按依赖顺序在 HTML 中加载
- **双模式运行**：Wails 桌面模式 + 浏览器 Mock 模式

### 视图结构（5 个面板）

```
index.html
├── 📂 工作区 (view-opencode)        ← 主视图
│   ├── 控制栏：启动/停止服务、网络配置、终端打开
│   ├── 左侧栏：项目→目录→会话树
│   ├── 中间栏：消息流（Markdown 渲染 + SSE 流式更新）
│   └── 右侧栏：服务状态 + 后台任务 + 待办事项 + 文件 diff
│
├── 🔌 供应商配置 (view-providers)   ← 增删改 API Key/Base URL
├── ⚙️ OMO 配置 (view-omo)           ← 模型按 agent/category 配置
├── 🔗 技能管理 (view-skills)        ← 符号链接 toggle
└── 📋 常用命令 (view-commands)      ← CLI/TUI 命令参考
```

### 事件流

```
OpenCode serve (/global/event)
  → service/sse.go (SSE 解析)
  → wruntime.EventsEmit("oc-event", data)
  → 前端 chat.js (监听 oc-event)
  → 按 type 分发：message/progress/tool/todo/diff...
  → DOM 增量更新
```

## 六、功能全景

| 功能模块 | 核心能力 |
|---------|---------|
| **工作区** | 一键启停 OpenCode Web Serve、代理配置、项目树导航、会话 CRUD、消息查看（Markdown 渲染 + 工具调用折叠）、SSE 流式增量更新、Agent/Model 选择器、/ 命令面板（summarize/revert/unrevert）、附件拖拽粘贴、右侧面板（状态/后台任务/待办/diff） |

| **常用命令** | CLI/TUI 命令参考（静态数据 + 从 OpenCode serve 动态获取） |

## 七、关键设计决策

1. **通用 API 代理**：`OpenCodeAPI(method, path, body)` 不限制端点，前端可调用任意 OpenCode API，后端无需逐一封装
2. **SSE 透传**：Go 后端只做解析和转发，不做消息处理，业务逻辑全在前端
3. **JSONC 行级编辑**：不序列化整个配置再写回，而是精确修改目标行，保留注释和格式
4. **Mock 双模式**：`api-mock.js` Proxy 模式让前端可脱离 Go 独立运行，开发调试方便
5. **进程树管理**：使用 `taskkill /F /T` 确保子进程（bun）也被清理
6. **原子写配置**：先写临时文件 → rename 替换，防止崩溃损坏配置
