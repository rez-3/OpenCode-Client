## 一、技术特点

- **零框架依赖**：纯原生 HTML/CSS/JS，无 React/Vue/Angular
- **模块化加载**：12 个 JS 文件按依赖顺序在 HTML 中加载
- **双模式运行**：Wails 桌面模式 + 浏览器 Mock 模式
- **JS 加载顺序**：`theme.js → utils.js → api-mock.js → navigation.js → chat.js → command-palette.js → omo-config.js → skill-manager.js → commands-view.js → provider-view.js → main.js`

## 二、核心模式：api-mock.js Proxy

前端与 Go 后端的通信通过一个 **JavaScript Proxy** 对象实现自动路由：

```javascript
const api = new Proxy({}, {
    get(_, prop) {
        if (window.go && window.go.main && window.go.main.App[prop]) {
            return window.go.main.App[prop];  // Wails 真实 API
        }
        return mockApi[prop];  // Mock 回退
    }
});
```

这使得前端可以**脱离 Go 后端在浏览器中独立预览**，Mock 数据覆盖了所有主要功能（技能列表、模型配置、供应商、命令参考、消息流、会话树等）。

## 五、消息渲染管道

### 5.1 数据流

```
SSE (oc-event)
  → handleOcEvent()          按 event.type 分发
  → upsertMessage/upsertPart/applyPartDelta   写入 messageCache
  → scheduleRenderCachedMessages()            requestAnimationFrame 防抖
  → renderMessages()         遍历缓存 → 生成 DOM
  → smartScroll()            智能滚动（用户手动滚动时停止自动滚动）
```

### 5.2 Part 渲染器

每条消息由多个 part 组成，`renderPart()` 按 type 分发：

| Part 类型 | 渲染函数 | 说明 |
|-----------|---------|------|
| `step-start` / `step-finish` | `renderStepDivider()` | 步骤分隔线（含 token 消耗） |
| `reasoning` | `renderReasoning()` | 思考过程（折叠/展开，Markdown 渲染） |
| `tool` | `renderTool()` | 工具调用（bash/file/tool 分类，状态 + 输入输出） |
| `text` | `renderTextPart()` | 文本（Markdown 渲染） |
| `file` | `renderFilePart()` | 附件文件（折叠/展开） |
| `patch` | `renderPatchPart()` | 文件补丁 |
| `agent` / `subtask` | `renderAgentPart()` | 代理/子任务（简略显示） |
| `compaction` | `renderCompaction()` | 上下文压缩标记 |
| `snapshot` | `renderSnapshot()` | 文件快照标记 |
| `retry` | `renderRetry()` | 重试标记 |

### 5.3 工具状态

工具 part 有四种状态，通过 CSS 类区分：

| 状态 | CSS 类 | 显示 |
|------|--------|------|
| 运行中 | `.running` | ⏳ 运行中（默认展开） |
| 已完成 | `.done` | ✓ 完成（默认折叠） |
| 失败 | `.error` | ✗ 失败 |
| 等待 | `.pending` | 等待 |

## 六、SSE 事件处理

### 6.1 事件流建立

```javascript
function startEventStream() {
    window.runtime.EventsOn('oc-event', (raw) => handleOcEvent(parseEventPayload(raw)));
    window.runtime.EventsOn('oc-event-error', (msg) => showToast('事件流异常: ' + msg, 'error'));
    if (api.StartOpenCodeEvents) api.StartOpenCodeEvents();
}
```

### 6.2 事件分发

| SSE 事件 | 处理逻辑 |
|----------|---------|
| `server.connected` / `server.heartbeat` | 忽略 |
| `permission.*` | Toast 权限请求提示 |
| `session.error` | 记录错误，当前会话重新加载 |
| `session.status` / `session.idle` | 更新发送按钮状态，idle 时重新加载消息并刷新树 |
| `message.updated` | `upsertMessage()` → 计划渲染 |
| `message.part.updated` | `upsertPart()` + `trackBackgroundTask()`（agent/subtask） |
| `message.part.delta` | `applyPartDelta()` + 流式增量更新后台任务输出 |
| `message.part.removed` | `removePart()` |
| `message.removed` | `removeMessage()` |
| `session.created` / `session.deleted` | 重建项目树 + 加载 diff |
| `session.updated` | 加载 diff + 防抖刷新树 |

## 七、后台任务模块（bgTaskSection）

> 后台任务模块为项目后来新增功能，详见对应设计文档。

### 7.1 数据模型

```javascript
let backgroundTasks = {};  // { taskId: TaskState }

TaskState = {
    id, name, type, status,      // 标识
    input, output, text, error,  // 内容
    messageId, sessionId,        // 关联
    startTime, endTime           // 计时
}
```

### 7.2 交互

- **自动追踪**：SSE 事件中的 `agent`/`subtask` part 自动记录
- **流式更新**：delta 增量实时追加输出
- **右侧面板**：独立卡片展示（名称 + 状态 + 耗时 + 输出预览）
- **折叠/展开**：点击卡片头部切换
- **跳转**：双击卡片跳转到消息流对应位置
- **生命周期**：切换/新建会话时自动清空

## 八、辅助模块

### 8.1 附件系统

- 支持文件拖拽、粘贴、点击选择
- 通过 `FileReader` 读取内容为 base64/dataURL
- SSE `message.updated` 完成后自动清空附件列表

### 8.2 搜索功能

- `/` 键或按钮打开搜索栏
- 在已渲染的消息 DOM 中全文搜索
- 支持临时展开折叠的 part（`temporarilyRevealSearchResult`）
- 搜索结果高亮 + 上下导航

### 8.3 主题系统

- 深色/浅色双主题
- CSS 变量驱动（`var(--bg-card)`, `var(--accent)` 等）
- `localStorage` 持久化
- 图标随主题切换（🌙 ↔ ☀️）

### 8.4 网络配置

- 弹窗配置 OpenCode 服务地址/端口 + HTTP 代理
- 代理配置以环境变量形式注入 `opencode serve` 进程
- `localStorage` 持久化（`NETWORK_CONFIG_KEY`）
