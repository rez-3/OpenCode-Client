## 一、技术特点

- **零框架依赖**：纯原生 HTML/CSS/JS，无 React/Vue/Angular
- **模块化加载**：12 个 JS 文件按依赖顺序在 HTML 中加载
- **双模式运行**：Wails 桌面模式 + 浏览器 Mock 模式
- **加载顺序**：`theme.js → utils.js → api-mock.js → navigation.js → chat.js → command-palette.js → omo-config.js → skill-manager.js → commands-view.js → provider-view.js → main.js`

---

## 二、核心模式：api-mock.js Proxy

```javascript
const api = new Proxy({}, {
    get(_, prop) {
        if (window.go && window.go.main.App[prop]) {
            return window.go.main.App[prop];  // Wails 真实 API
        }
        return mockApi[prop];  // Mock 回退
    }
});
```

前端可脱离 Go 后端在浏览器中独立预览，Mock 覆盖所有主要功能。

## 五、消息渲染管道

```
SSE (oc-event) → handleOcEvent() → 按 event.type 分发
  → upsertMessage / upsertPart / applyPartDelta → 写入 messageCache
  → scheduleRenderCachedMessages() → requestAnimationFrame 防抖
  → renderMessages() → 遍历缓存 → 生成 DOM
  → smartScroll() → 智能滚动（手动拖拽时停止自动跟随）
```

### Part 类型

| Part 类型 | 说明 |
|-----------|------|
| `step-start` / `step-finish` | 步骤分割线（含 token 消耗） |
| `reasoning` | 思考过程（折叠/展开） |
| `tool` | 工具调用（bash/file/tool，状态 + 输入输出） |
| `text` | 文本（Markdown 渲染） |
| `file` / `patch` | 附件文件、文件补丁 |
| `agent` / `subtask` | 代理 / 子任务 |
| `compaction` / `snapshot` / `retry` | 系统标记 |

## 六、SSE 事件分发

| SSE 事件 | 处理 |
|----------|------|
| `session.status` | 更新发送按钮状态；当前会话 idle 时重新加载消息，不刷新树 |
| `session.idle` | 清理错误状态、更新发送按钮、重新加载消息，不刷新树 |
| `message.updated` | upsertMessage → 调度渲染 |
| `message.part.updated` | upsertPart + trackBackgroundTask |
| `message.part.delta` | applyPartDelta + 流式增量更新 |
| `message.part.removed` / `message.removed` | removePart / removeMessage |
| `session.created`（仅命中当前主会话） / `session.deleted` | 重建项目树 + 加载 diff |
| `session.updated` | 刷新 diff，不刷新项目树 |

当前实现中，项目树采用 `buildTree()` + `api.GetProjectTree()` 的**按需全量重建**策略，只在显式主会话创建命中、会话删除或显式操作时触发，不跟随消息流事件持续刷新；回复过程中由子任务派生出的 `session.created` 不会直接刷新项目树。

## 七、子任务面板（右侧面板）

跟踪后台 agent/subtask 执行状态：

- SSE 事件中的 `agent`/`subtask` part 自动记录
- 流式更新 delta 增量追加输出
- 卡片展示：名称 + 状态 + 耗时 + 输出预览
- 点击卡片跳转到消息流对应位置
- 详情弹窗查看完整子任务消息
