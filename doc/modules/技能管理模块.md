# 技能管理模块

> 返回 [项目架构深度分析](../项目架构深度分析.md)

---

## 一、模块定位

`skill/` 包负责管理 OpenCode 的 **技能（Skill）** 系统。技能是 OpenCode 的扩展机制，每个技能是一个包含 `SKILL.md` 文件的目录，通过**符号链接**挂载到 `~/.config/opencode/skills/` 下。

---

## 二、核心结构

```go
type Manager struct {
    globalDir string  // ~/.config/opencode/skills/
}

func NewManager() *Manager {
    homeDir, _ := os.UserHomeDir()
    return &Manager{
        globalDir: filepath.Join(homeDir, ".config", "opencode", "skills"),
    }
}
```

`Manager` 被 `App` 结构体持有，在 `NewApp()` 时创建，`Refresh()` 时重建。

## 三、技能扫描（GetAllSkills）

### 3.1 扫描逻辑

`scanDir(dir, source, prefix)` 递归扫描：

1. `os.ReadDir(dir)` 获取目录条目
2. 对每个条目，检查是否存在 `SKILL.md`（`os.Stat` 会跟随 Junction）
3. 有 SKILL.md → 解析为技能，提取名称和描述
4. 无 SKILL.md 但是目录/Junction → 递归扫描子目录
5. 构建相对名称（如 `superpowers/brainstorming`）

### 3.2 链接检测

```go
if info, err := os.Lstat(linkPath); err == nil {
    linked = (info.Mode()&os.ModeSymlink != 0) || info.IsDir()
}
```

`os.Lstat` 不跟随符号链接，用于判断条目是真实目录还是符号链接。

### 3.3 Frontmatter 解析

手动解析 YAML frontmatter（`---` 包裹区）：

```yaml
---
name: brainstorming
description: 设计开发前的头脑风暴分析
---
```

提取 `name` 和 `description` 字段，支持双引号包裹值。

## 四、链接切换（ToggleSkill）

### 4.1 启用（enable = true）

```go
if runtime.GOOS == "windows" {
    cmd := exec.Command("cmd", "/c", "mklink", "/J", linkPath, skillPath)
    if err := cmd.Run(); err != nil {
        // 回退到 os.Symlink
        os.Symlink(skillPath, linkPath)
    }
} else {
    os.Symlink(skillPath, linkPath)
}
```

**Windows 双策略**：
1. 优先 `mklink /J`（目录 Junction，无需管理员权限）
2. 失败时回退 `os.Symlink`（需开发者模式或管理员权限）

### 4.2 禁用（enable = false）

直接 `os.Remove(linkPath)` 删除符号链接。

## 六、跨平台兼容

| 平台 | 符号链接实现 | 说明 |
|------|------------|------|
| Windows | `mklink /J` → `os.Symlink` 回退 | Junction 更可靠，Symlink 需特定权限 |
| macOS | `os.Symlink` | 标准 Unix 符号链接 |
| Linux | `os.Symlink` | 标准 Unix 符号链接 |

## 七、与 App 的集成

```go
// app.go
type App struct {
    ctx context.Context
    sm  *skill.Manager  // 持有技能管理器
}

func (a *App) GetSkills() []model.SkillInfo {
    return a.sm.GetAllSkills()  // 委托扫描
}

func (a *App) ToggleSkill(path, name string, enable bool) model.ToggleResult {
    newState, err := a.sm.ToggleSkill(path, name, enable)
    // 包装为 ToggleResult 返回
}

func (a *App) Refresh() error {
    a.sm = skill.NewManager()  // 重建管理器以刷新
    return nil
}
```

刷新策略是**重建整个 Manager**，因为技能目录可能发生了增删变化。这是一个轻量级操作（仅重新扫描目录）。

## 八、前端视图

技能管理视图（`skill-manager.js`）提供：

- 统计栏：全局技能数量
- 源路径：显示 `~/.config/opencode/skills/`，可点击在资源管理器中打开
- 搜索栏：按名称即时过滤
- 技能列表：名称 + 描述 + 路径 + 链接开关 toggle
- 技能预览/编辑 Modal：查看和编辑 SKILL.md 内容
- 批量操作：全部启用/禁用
