# 技能管理模块

> 返回 [项目架构深度分析](../项目架构深度分析.md)

---

## 一、模块定位

技能管理模块负责 OpenCode 技能的扫描、符号链接启停、技能文件浏览和文本编辑。当前“打开”行为已统一为站内技能文件浏览/编辑弹窗，不再区分桌面端和 Web 端的资源管理器行为。

详见 [技能文件浏览编辑模块](技能文件浏览编辑模块.md)。

---

## 二、核心结构

后端依赖 `skill/skill.go`：

- `GetAllSkills()` 扫描技能目录
- `ToggleSkill()` 启停符号链接 / Junction
- `ReadSkillContent()` / `SaveSkillContent()` 读写 SKILL.md
- `ListSkillFiles()` / `ReadSkillFile()` / `SaveSkillFile()` 提供技能文件浏览/编辑

前端依赖 `frontend/dist/skill-manager.js`：

- 技能列表渲染
- 搜索过滤
- 统一“打开”弹窗
- 文件树、高亮、主文件标识、Markdown 预览、编辑保存

---

## 三、技能扫描

`scanDir(dir, source, prefix)` 递归扫描目录：
1. `os.ReadDir(dir)` 获取目录条目
2. 对每个条目检查是否存在 `SKILL.md`
3. 有 SKILL.md 则视为技能目录，提取名称与描述
4. 无 SKILL.md 但仍为目录 / Junction 时继续递归

这条扫描链路兼容 Windows Junction。与之相关的真实文件浏览/保存链路已单独做了兼容与边界控制。

---

## 四、统一的技能文件浏览/编辑

### 打开行为

- 技能名点击：进入站内浏览器
- “打开”按钮：进入同一弹窗
- “详情”按钮：已移除

### 弹窗能力

- 左侧目录树
- 当前选中文件高亮
- `SKILL.md` 置顶并带“主文件”标识
- Markdown 预览
- 文本文件编辑与保存
- 未保存修改保护

### 安全边界

- 仅允许访问技能根目录内的文件
- 仅允许编辑文本文件
- 技能目录本身可为 Junction/软链接
- 内部路径仍受技能根目录边界限制

---

## 五、桌面端与 Web 端差异

| 能力 | 桌面端 | Web 端 |
|------|--------|--------|
| 全局目录打开 | 保留 | 隐藏 |
| “打开”技能 | 站内浏览/编辑弹窗 | 站内浏览/编辑弹窗 |
| 保存技能文件 | 支持 | 支持 |

也就是说，技能文件浏览器已经实现了两端统一交互，而不是桌面端系统打开、Web 端站内预览的分裂模式。
