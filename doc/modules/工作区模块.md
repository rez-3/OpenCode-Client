## 一、概述

工作区是整个应用的主视图，提供 OpenCode serve 管理、项目树、会话消息流、服务状态、子任务、代办和文件变更等功能。当前工作区已扩展出三条能力线：

- **桌面端工作区**：系统目录选择器、系统目录打开、完整三栏布局
- **浏览器工作区**：页面 Web 服务、`app-call`、站内目录浏览器
- **手机端工作区**：项目树左抽屉、隐藏 toolbar/sidebar/sidepanel、长会话窗口化

## 三、服务管理

工作区服务管理仍由 `service/process.go` 和 `service/sse.go` 支撑，负责：

- 启动/停止 OpenCode serve
- 检查 `/global/health`
- 透传 `/global/event` SSE
- 维护 LastCfgHost / LastCfgPort

页面 Web 服务本身见 [页面 Web 服务模块](页面Web服务模块.md)。

## 四、项目树

### 数据结构

项目 (project) → 工作目录 (directory) → 会话 (session) 三级树。

### 构建流程

`GetProjectTree(knownDirs)`：
1. 从 `/project` 获取项目列表
2. 从 `/session?all=true&roots=true&limit=500` 获取会话
3. 对每个 `knownDirs` 目录单独查询会话
4. `buildTreeJSON()` 组装为项目/目录/会话树
5. 前端 `buildTree()` 全量重绘项目树

### 添加目录

| 模式 | 行为 |
|------|------|
| 桌面端 | 项目标签旁 `＋` → 系统目录选择器 → 写入 `oc-known-dirs` |
| Web 端 | 项目标签旁 `＋` → 站内目录浏览器 → 逐级选择目录 → 写入 `oc-known-dirs` |

Web 端站内目录浏览器详见 [Web 目录浏览器模块](Web目录浏览器模块.md)。

## 五、会话管理与消息渲染

### 会话切换

当前工作区已改成：
- 先显示 loading
- 再后台异步 `loadMessages()`
- 手机端点击会话后立即收起项目树抽屉

### 消息渲染

- 完整消息缓存仍保存在 `messageCache[currentSessionId]`
- 消息更新主要通过 SSE 改写缓存后再渲染
- 手机端长会话默认只渲染最近 30 条，顶部点击“加载更多”再展开更早消息

消息窗口化细节见 [消息窗口化与加载更多模块](消息窗口化与加载更多模块.md)。

手机端：
- 默认隐藏整个 `oc-sidepanel`
- 不做第二层抽屉

## 七、输入区

### 发送时序

为降低长会话卡顿：
- 手机端发送前只插入轻量 pending 占位
- 手机端发送后不立即全量 `loadMessages()`，依赖 SSE 增量回填
- 输入焦点期间暂停 `refreshTimer`

桌面端与大屏 Web 仍保留较完整的右侧信息流刷新逻辑。
