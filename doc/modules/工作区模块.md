## 一、概述

工作区是整个应用的主视图，提供 OpenCode Web Serve 的完整管理界面。包含服务启停、会话 CRUD、消息流查看、子任务追踪、文件变更 diff 等功能。

**后端**：`service/process.go`（进程管理）、`service/api.go`（API 代理）、`service/sse.go`（事件流）、`service/tree.go`（项目树）

**前端**：`chat.js`（~2700 行）、`main.js`（初始化绑定）、`navigation.js`（侧边栏）

## 三、服务管理

### 3.1 启动流程

```
用户点击 ▶ 启动
  → 读取配置（serviceHost / servicePort / proxy）
  → 检查已有进程：若 WebSess 非 nil，返回当前状态
  → 检查外部运行：isOpenCodeServerRunning(hostname, port)
  → 检查端口占用：isPortInUse(hostname, port)
  → 构建命令：opencode serve --port X --hostname Y
  → 注入环境变量：HTTP_PROXY / HTTPS_PROXY / ALL_PROXY / NO_PROXY
  → cmd.Start() → 轮询 TCP 端口（250ms × 40 次 = 10s 超时）
  → 返回健康状态 + 版本号
```

### 3.2 停止流程

```
用户点击 ■ 停止
  → 停止 SSE 事件流
  → taskkill /F /T /PID（Windows 进程树终止）
  → 检查端口占用回退清理
```

### 3.3 健康检查

`getOpenCodeHealth(hostname, port)` 请求 `/global/health`：
- 2 秒超时 HTTP GET
- 解析 version 字段
- 返回状态：在线/离线/异常/未知

### 3.4 自动检测

`getWebSession()` 在 WebSess 为空时，自动用 LastCfgHost:LastCfgPort 检测外部运行的 serve。用户手动启动 `opencode serve --port 4096` 后无需应用内按钮重新启动。

### 3.5 网络配置

弹窗配置服务地址和 HTTP 代理，持久化到 `localStorage`：

| 字段 | 默认值 | 说明 |
|------|--------|------|
| serviceHost | `127.0.0.1` | OpenCode 服务地址 |
| servicePort | `4096` | 服务端口 |
| proxyEnabled | `false` | HTTP 代理开关 |
| proxyHost | `127.0.0.1` | 代理地址 |
| proxyPort | `7897` | 代理端口 |

### 3.6 终端打开

`LaunchWindowsTerminal(mode, url, dir)`：
- 优先 `wt.exe`（Windows Terminal）
- 回退 `cmd /c start opencode`
- 支持 attach 模式（连接远程）和本地模式

## 四、项目树

### 4.1 数据结构

项目 (project) → 工作目录 (directory) → 会话 (session) 三级树。

### 4.2 构建流程

`GetProjectTree(knownDirs)`：
1. 从 `/project` 获取项目列表
2. 从 `/session?all=true&roots=true&limit=500` 获取全部会话
3. 对每个已知目录单独查询 `/session?directory=X` 获取 global 项目会话
4. 按 projectID → directory 两级分组
5. 按 updatedAt 排序

### 4.3 交互

| 操作 | 实现 |
|------|------|
| 加载树 | `buildTree()` → 渲染 HTML 树形结构 |
| 添加目录 | 项目标签旁 `＋` → 选择目录 → 写入 `oc-known-dirs` localStorage |
| 刷新 | `btnRefreshTree` → `buildTree()` |
| 展开/折叠 | `▼`/`▶` 切换 `.oc-tree-children` 显隐 |
| 切换会话 | `selectSession(id)` → 加载消息 → 更新右侧面板 |

## 五、会话管理

### 5.1 CRUD

| 操作 | 实现 | 说明 |
|------|------|------|
| 新建 | `createNewSession()` → 选择目录 → 设置 `pendingWorkDir` | 首次输入时才真正创建 |
| 创建 | `CreateSession(workDir)` → `POST /session` + `x-opencode-directory` header | body 为 `{}` |
| 命名 | 首次发送后 `PATCH /session/:id` 用输入前 15 字符作为标题 | |
| 删除 | `DELETE /session/:id` → 刷新树 | hover 显示 `✕` 按钮 |
| 切换 | `selectSession(id)` → `loadMessages()` → 强制滚底 | 右侧面板同步更新 |

### 5.2 工作目录

- OpenCode 实例目录优先级：`directory` query → `x-opencode-directory` header → `process.cwd()`
- 服务启动目录默认为程序 exe 所在目录
- 会话目录通过 `x-opencode-directory` 请求头在 `POST /session` 时设置

## 六、消息区

### 6.1 消息渲染管道

```
SSE (oc-event)
  → handleOcEvent()              按 event.type 分发
  → upsertMessage / upsertPart   写入 messageCache[sessionID]
  → scheduleRenderCachedMessages()                  RAF 防抖
  → renderMessages()              遍历缓存 → 生成 DOM
  → smartScroll()                 智能滚动（手动拖拽时停止跟随）
```

### 6.2 Part 渲染器

每条消息由多个 part 组成，`renderPart()` 按 type 分发：

| Part 类型 | 渲染函数 | 说明 |
|-----------|---------|------|
| `step-start` / `step-finish` | `renderStepDivider()` | 步骤分离线（含 token 消耗） |
| `reasoning` | `renderReasoning()` | 思考过程（折叠/展开，Markdown 渲染） |
| `tool` | `renderTool()` | 工具调用：bash / file / task，状态 + 输入输出卡片 |
| `text` | `renderTextPart()` | 文本（Markdown 渲染） |
| `file` | `renderFilePart()` | 附件文件 |
| `patch` | `renderPatchPart()` | 文件补丁 |
| `agent` / `subtask` | `renderAgentPart()` | 代理 / 子任务（简略视图） |
| `compaction` / `snapshot` / `retry` | — | 系统标记 |

### 6.3 工具状态

| 状态 | CSS 类 | 显示 |
|------|--------|------|
| 运行中 | `.running` | ⏳ 运行中（默认展开） |
| 已完成 | `.done` | ✓ 完成（默认折叠） |
| 失败 | `.error` | ✗ 失败 |
| 等待 | `.pending` | 等待 |

### 6.4 性能优化

| 优化 | 实现 |
|------|------|
| 事件批处理 | `scheduleRenderCachedMessages()` 用 `requestAnimationFrame` 合并高频事件 |
| 增量更新 | 流式输出时仅追加新增片段，不全量重建 DOM |
| 消息缓存 | `messageCache[sessionID]` 按会话隔离，事件驱动增量更新 |
| 滚动守卫 | `userScrolling` 标志跟踪拖拽，拖拽时跳过 DOM 重建 |
| CSS 隔离 | `.oc-messages` 使用 `contain: paint style` |

### 6.5 滚动行为

| 场景 | 行为 |
|------|------|
| 切换会话 | `smartScroll(box, true)` → 强制滚底 |
| 发送消息 | 乐观渲染 → 滚底 → 加载响应 → 再滚底 |
| 流式输出 | 根据 `nearBottom`（120px 阈值）决定是否跟随 |
| 手动拖拽 | `userScrolling` 守卫阻止自动滚动 |
| 点击回底 | `lastElementChild.scrollIntoView` |

## 七、SSE 事件流

### 7.1 架构

```
OpenCode serve → /global/event SSE
  → service/sse.go 逐行解析 data: 前缀
  → wruntime.EventsEmit("oc-event", data)
  → 前端 window.runtime.EventsOn("oc-event", handleOcEvent)
  → 按类型分发到各处理函数
```

### 7.2 事件分发

| 事件 | 处理 |
|------|------|
| `session.status` | 更新 busy/idle 状态，idle 时刷新消息和树 |
| `session.error` | 记录错误，当前会话重新加载 |
| `message.updated` | upsertMessage → 计划渲染 |
| `message.part.updated` | upsertPart + 子任务追踪 |
| `message.part.delta` | 增量追加 part 内容 |
| `message.part.removed` | 移除 part |
| `message.removed` | 移除消息 |
| `session.created` / `session.deleted` | 重建项目树 |
| `session.updated` | 刷新 diff |
| `server.connected` / `server.heartbeat` | 忽略 |
| `permission.*` | Toast 权限请求提示 |

### 7.3 重连

前端在 `oc-event-error` 时触发 `checkWebStatus()` 重新建立连接。

## 八、右侧面板

### 8.1 子任务面板

跟踪 session 中的后台 task 执行状态：

- SSE `agent`/`subtask` part 自动记录
- 流式 delta 实时追加输出
- 卡片展示：名称 + 状态徽标 + 耗时 + 输出预览
- 点击卡片 → 跳转到对应主消息
- 详情按钮 → Modal 弹窗加载子任务完整消息

数据模型：
```javascript
let subtaskSummaries = [];  // { childSessionId, title, agent, model, status, durationMs, ... }
```

### 8.2 服务状态

显示 OpenCode 服务 URL、健康状态（在线/离线/异常）、版本号。

通过 `GetWebStatus()` → `/global/health` 获取。

### 8.3 代办事项

从当前会话的 assistant 消息中提取最后一次 `todowrite` 工具调用的 todos：

| 显示 | 逻辑 |
|------|------|
| 进行中 | 按优先级色条（高=红、中=黄、低=灰）分组 |
| 已完成 | 降低透明度 + 删除线 |
| 全部完成 | 显示"会话中暂无代办" |

### 8.4 文件变更

显示当前会话的文件变更 diff，通过 `GET /session/:id/diff` 获取，树形展示文件增减。

## 十一、线程安全

所有涉及 `WebSess` 的操作通过 `WebSessMu sync.Mutex` 保护。启动采用先 Lock 检查、再 Unlock 执行、再 Lock 赋值的两段锁模式。
