# 供应商配置模块（view-providers）

> 返回 [项目架构深度分析](../项目架构深度分析.md)

---

## 一、概述

管理 AI 模型供应商（DeepSeek、OpenAI、Anthropic、Ollama 等本地模型）。表格化编辑供应商名称、baseURL、API Key 和模型列表。

**后端**：`config/provider_config.go`（~170 行）

**前端**：`provider-view.js`（~180 行）

---

## 二、配置文件

- **路径**：`~/.config/opencode/opencode.jsonc`
- **Wails 绑定**：`GetProviderConfigPath()` 返回路径

```jsonc
{
    "$schema": "...",
    "provider": {
        "deepseek": {
            "npm": "@ai-sdk/openai-compatible",
            "name": "DeepSeek",
            "options": {
                "baseURL": "https://api.deepseek.com/v1",
                "apiKey": "sk-xxx"
            },
            "models": {
                "deepseek-v4-pro": { "name": "DeepSeek-V4-Pro" },
                "deepseek-v4-flash": { "name": "DeepSeek-V4-Flash" }
            }
        }
    },
    "enabled_providers": ["deepseek"]
}
```

## 四、核心功能

### 4.1 读取

```javascript
// provider-view.js
async function loadProviders() {
    const [providers, cfgPath] = await Promise.all([
        api.GetProviders(),
        api.GetProviderConfigPath(),
    ]);
    document.getElementById('providerConfigPath').textContent = cfgPath || '未知';
    renderProviders(providers || []);
}
```

### 4.2 新增

点击底部虚线"添加供应商"卡片 → 新增空卡片 → 填写 key / name / baseURL / API Key → 添加模型 → 点击保存。

### 4.3 编辑

直接修改已有卡片中的字段 → 点击保存。

### 4.4 删除

点击 `✕` → 确认 → `api.DeleteProvider(key)`。

### 4.5 API Key 明文切换

点击 `👁` 按钮在 `type="password"` 和 `type="text"` 之间切换。

### 4.6 模型中开关

`Enabled` 通过 checkbox 控制 → 保存时更新 `enabled_providers` 列表。

### 4.7 保存

```javascript
// 从 DOM 收集当前卡片数据 → 调用后端
api.SaveProvider({
    key, name, baseURL, apiKey, enabled, models
})
```

## 六、设计特点

- **完整序列化**：与模型配置的行级编辑不同，供应商配置使用标准 `json.Marshal` / `json.Unmarshal`
- **自动默认值**：`SaveProvider` 自动设置 `npm: "@ai-sdk/openai-compatible"` 和 `setCacheKey: true`
- **Enabled 分离**：启用状态通过 `enabled_providers` 列表管理，而非 provider 内部字段
