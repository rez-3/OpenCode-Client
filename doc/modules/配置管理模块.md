## 一、模块定位

`config/` 包负责 **OpenCode 配置文件**的读写操作，直接操作 `~/.config/opencode/` 下的 JSONC 文件。这是整个项目中最复杂的模块（~755行），核心难点在于**保留 JSONC 注释和格式的同时进行精确的增删改**。

| `provider_config.go` | 169 | opencode.jsonc 的供应商配置读写 |

## 三、模型配置（model_config.go）

### 3.1 配置文件

- **路径**：`~/.config/opencode/oh-my-openagent.jsonc`（优先 .jsonc，不存在时回退 .json）
- **结构**：按 agent/category 分组，每项包含 model 字段

```jsonc
{
    // 执行者：负责执行具体任务
    "agents": {
        "sisyphus": { "model": "deepseek/deepseek-v4-pro" },
        "oracle": { "model": "deepseek/deepseek-v4-flash" }
    },
    "categories": {
        "quick": { "model": "deepseek/deepseek-v4-flash" },
        "visual-engineering": { "model": "deepseek/deepseek-v4-flash" }
    }
}
```

### 3.2 核心功能

| 操作 | 方法 | 说明 |
|------|------|------|
| 读取 | `LoadConfig()` | 解析 JSONC → OpenAgentConfig + 原始文本 + 注释映射 |
| 保存 | `SaveConfig(entries)` | 批量更新 model 值（只改行内 model 字段，不重建整段） |
| 新增条目 | `AddConfigEntry(key, model, type)` | 在指定 section 末尾插入新条目 |
| 删除条目 | `DeleteConfigEntry(key, type)` | 精确删除指定 key 的对象块 |
| 新增类型 | `AddModelType(type)` | 在根对象末尾插入新的空 section |
| 删除类型 | `DeleteModelType(type)` | 删除整个 section 块 |
| 全量读写 | `GetFullConfig()` / `SaveFullConfig(json)` | 前端直接修改完整 JSON 字符串 |
| 转换 | `ConfigToEntries(config, comments)` | 将 Go 结构体转为前端展示用的 ModelEntry 列表 |

### 3.3 注释保留算法

这是模块最精妙的部分——不序列化整个 JSON 再写回（会丢失注释和格式），而是**行级精确编辑**：

1. **读取时**：`extractComments()` 逐行扫描，匹配 `"key": {` 后的 `// 注释` 文本
2. **保存时**：`SaveConfig()` 逐行遍历原始文本，用正则 `("model"\s*:\s*)"[^"]*"` 定位并替换 model 值
3. **新增时**：`insertModelEntry()` 在 section 闭合 `}` 前插入新行，带 `// 注释` 格式
4. **删除时**：`removeModelEntry()` 找到 `"key": { ... }` 完整块并移除，自动清理尾部逗号

## 四、供应商配置（provider_config.go）

### 4.1 配置文件

- **路径**：`~/.config/opencode/opencode.jsonc`
- **结构**：provider 字典 + enabled_providers 列表 + 其他元数据

```jsonc
{
    "$schema": "...",
    "provider": {
        "deepseek": {
            "npm": "@ai-sdk/openai-compatible",
            "name": "DeepSeek",
            "options": { "baseURL": "https://api.deepseek.com/v1", "apiKey": "sk-xxx" },
            "models": { "deepseek-v4-pro": { "name": "DeepSeek-V4-Pro" } }
        }
    },
    "enabled_providers": ["deepseek"]
}
```

### 4.2 核心功能

| 操作 | 方法 | 说明 |
|------|------|------|
| 读取 | `GetProviders()` | 解析 → 前端展示的 ProviderInfo 列表（含 baseURL/apiKey/models） |
| 保存 | `SaveProvider(ps)` | 新增或更新单个供应商，自动管理 enabled_providers |
| 删除 | `DeleteProvider(key)` | 删除供应商，同时从 enabled_providers 中移除 |

### 4.3 设计特点

- 供应商配置使用**完整的 JSON 序列化/反序列化**（与模型配置的行级编辑不同），因为结构相对固定
- `SaveProvider` 自动设置 `npm: "@ai-sdk/openai-compatible"` 和 `setCacheKey: true`
- `Enabled` 字段通过 enabled_providers 列表管理，而非 provider 内部字段

## 五、两者对比

| 维度 | 模型配置 | 供应商配置 |
|------|---------|-----------|
| 文件 | oh-my-openagent.jsonc | opencode.jsonc |
| 编辑策略 | 行级精确编辑（保留注释） | 完整序列化/反序列化 |
| 复杂度 | 高（755行，注释提取+行级增删） | 低（169行，标准 JSON 操作） |
| 为什么不同 | 用户手写注释需保留 | 结构固定，注释较少 |
