# 技能文件浏览编辑模块

> 返回 [技术方案](../技术方案.md)

---

## 一、模块定位

统一桌面端和 Web 端技能管理中的“打开”行为：不再走系统资源管理器，而是进入站内技能文件浏览/编辑弹窗。默认预览，支持目录树、当前选中文件高亮、`SKILL.md` 主文件标识、Markdown 预览、文本文件编辑与保存。

---

## 二、前端职责

主要集中在 `frontend/dist/skill-manager.js`：

- 打开技能文件浏览弹窗
- 目录树渲染与折叠
- 当前文件高亮
- `SKILL.md` 优先显示与主文件标识
- 文本 / Markdown 预览
- 编辑模式切换与保存
- 未保存变更保护

样式集中在 `frontend/dist/style.css`。

---

## 三、后端职责

主要集中在 `skill/skill.go` 与 `app.go`：

| 方法 | 说明 |
|------|------|
| `ListSkillFiles(skillPath)` | 返回技能根目录树 |
| `ReadSkillFile(skillPath, relativePath)` | 读取技能根目录内文本文件 |
| `SaveSkillFile(skillPath, relativePath, content)` | 保存技能根目录内文本文件 |

这些方法通过 `app.go` 暴露给 `AppCall`，再由浏览器模式的 `api-mock.js` 通用 RPC 回退使用。

---

## 四、安全边界

1. 只允许访问技能根目录内的内容
2. 保存仅支持文本文件
3. `SKILL.md` 默认优先展示
4. 技能目录本身可为 Junction/软链接；但内部路径仍受技能根目录边界约束

---

## 五、用户体验细节

- 技能名点击和“打开”按钮都进入同一弹窗
- “详情”按钮已移除
- `SKILL.md` 在目录树最上方，并带“主文件”标识
- 当前选中文件高亮明显
- Markdown 文件用 `marked` 渲染预览
- 编辑时支持未保存变更确认
