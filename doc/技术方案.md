# OpenCode 管理中心 — 技术方案

## 项目定位

Wails v2 桌面应用（Go 后端 + 原生 JS 前端），为 OpenCode AI 编程助手提供可视化管理中心。

---

## 一、技术栈

| 维度 | 技术 |
|------|------|
| 桌面框架 | Wails v2.12 |
| 后端 | Go 1.25 |
| 前端 | 原生 HTML/CSS/JS（零框架） |
| Markdown | marked.js |
| 浏览器 | WebView2（Windows） |
| API 代理 | Go `net/http` 透传 |
| 进程管理 | `os/exec` + TCP 端口探测 |

---

## 二、模块总览

应用包含 5 个 Tab 页面，每个 Tab 有独立的功能模块文档：

| Tab | 视图 ID | 功能 | 文档 |
|-----|---------|------|------|
| 📂 工作区 | `view-opencode` | OpenCode 服务管理、会话 CRUD、消息流、子任务追踪、项目树 | [工作区模块](modules/工作区模块.md) |
| 🔌 供应商配置 | `view-providers` | AI 供应商的增删改、API Key 和模型管理 | [供应商配置模块](modules/供应商配置模块.md) |
| ⚙️ OMO 配置 | `view-omo` | 模型配置编辑、本地方案导入/导出/入库/切换 | [OMO 配置模块](modules/OMO配置模块.md) |
| 🔗 技能管理 | `view-skills` | 技能符号链接的扫描、启用/禁用、内容编辑 | [技能管理模块](modules/技能管理模块.md) |
| 📋 常用命令 | `view-commands` | CLI/TUI 命令参考（静态 + 动态） | [常用命令模块](modules/常用命令模块.md) |

架构分析见 [项目架构深度分析](项目架构深度分析.md)。

## 三、架构分层

```
┌──────────────────────────────────────┐
│        前端（WebView2）               │
│  原生 JS · 12 文件模块化 · CSS 变量    │
│  双模式：Wails 桌面 / 浏览器 Mock      │
├──────────────────────────────────────┤
│     Wails Runtime Bridge              │
│  Go 方法 → JS 绑定 (window.go.main)   │
├──────────────────────────────────────┤
│         Go 后端（App 门面）            │
│  app.go → config / service / skill    │
└──────────────────────────────────────┘
```

---

## 四、关键文件

| 文件 | 职责 |
|------|------|
| `main.go` | Wails 入口，embed 前端，生命周期钩子 |
| `app.go` | App 门面，所有前端绑定方法（~370 行） |
| `web.go` | OpenCode serve 启动/停止/API 代理 |
| `config/model_config.go` | oh-my-openagent.jsonc 行级 model 值替换 |
| `config/agent_descriptions.go` | agent/category 描述表读写 |
| `config/provider_config.go` | opencode.jsonc 供应商配置 |
| `service/process.go` | 进程管理 + 健康检查 |
| `service/scheme.go` | 本地方案读写、导出 |
| `model/types.go` | 前后端共享数据结构 |
| `frontend/dist/chat.js` | 工作区核心（~2700 行） |
| `frontend/dist/omo-config.js` | OMO 配置 + 方案管理（~660 行） |
| `frontend/dist/style.css` | 样式表（~3250 行） |
| `configs/oh-my-openagent/agents-comments.json` | 描述文件 |
| `.sisyphus/omo-schemes/` | 本地方案存储 |
