# OpenCode 管理中心 — 技术方案

## 项目定位

Wails v2 桌面应用（Go 后端 + 原生 JS 前端），为 OpenCode AI 编程助手提供可视化管理中心。当前系统已从单纯桌面应用扩展为：

- 桌面端 Wails 运行模式
- 浏览器可访问的页面 Web 服务
- Web 端目录浏览器与技能文件浏览/编辑能力
- 手机端工作区专用布局与长会话窗口化策略

---

## 一、技术栈

| 维度 | 技术 |
|------|------|
| 桌面框架 | Wails v2.12 |
| 后端 | Go 1.25 |
| 前端 | 原生 HTML/CSS/JS（零框架） |
| Markdown | marked.js |
| 浏览器 | WebView2（Windows） |
| API 代理 | Go `net/http` 透传 + `app-call` 通用 RPC |
| 进程管理 | `os/exec` + TCP 端口探测 |

## 三、架构分层

```text
前端（WebView2 / 浏览器）
  ├─ 原生 JS / CSS 变量主题 / 目录浏览器 / 技能文件浏览器
  ├─ 手机端抽屉布局 / 长会话窗口化渲染
  └─ SSE 事件驱动消息缓存与增量更新

Wails Runtime Bridge / Web app-call RPC

Go 后端（App 门面）
  ├─ config/   配置读写
  ├─ service/  进程管理、前端 Web 服务、目录浏览器、树与 API 代理
  ├─ skill/    技能扫描、链接管理、文件浏览与保存
  └─ model/    前后端共享类型
```

工作区采用 **SSE 事件驱动 + 前端消息缓存/增量渲染**。项目树仍由前端 `buildTree()` 发起，全量获取树结构，但会话消息渲染已增加移动端窗口化与加载更多策略。

## 四、关键文件

| 文件 | 职责 |
|------|------|
| `main.go` | Wails 入口，embed 前端资源，生命周期钩子 |

| `frontend/dist/skill-manager.js` | 技能管理：技能树、文件浏览/编辑、Markdown 预览 |
| `frontend/dist/omo-config.js` | OMO 配置、方案管理 |
| `frontend/dist/style.css` | 全局样式与响应式布局 |
| `test/frontend/frontend_web_test.go` | 页面 Web 服务协议层测试 |
| `test/service/skill_symlink_test.go` | 技能软链接 / Junction 路径兼容测试 |

## 六、构建与执行约束

项目使用 Wails CLI 构建，`wails.json` 位于**项目根目录**。因此构建命令必须在仓库根目录执行，例如：

```powershell
wails build
```

如果在 `build/bin` 等子目录执行，会因为找不到 `wails.json` 而直接失败。这属于执行目录问题，不是源码编译错误。
