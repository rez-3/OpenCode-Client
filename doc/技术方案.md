# OpenCode 管理中心 — 技术方案

## 整体架构

Wails v3 桌面应用（Go 后端 + 原生 JS 前端），通过 HTTP 代理 OpenCode Web Serve API，提供可视化管理界面。

```
┌──────────────────────────────────────────────────────┐
│  顶部导航栏（标题 + 主题切换）                          │
├────────┬──────────────────────────┬──────────────────┤
│ 侧边栏  │      中间（会话区）        │   右侧面板        │
│        │                          │                  │
│ 📂 工作区│  项目树 │ 消息列表         │ 服务状态（可折叠） │
│ 🔌 供应商│  └ 项目 │  └ 角色+内容     │ 代办事项（可折叠） │
│ ⚙️ 模型 │    └ 目录│    └ 工具调用     │ 修改（可折叠）    │
│ 🔗 技能 │      └ 会话│  └ 推理过程     │ 📂 工作目录      │
│ 📋 命令 │                          │                  │
├────────┴──────────────────────────┴──────────────────┤
│  输入区（文字 + 附件 + 发送/停止按钮）                   │
└──────────────────────────────────────────────────────┘
```

## 模块 1：工作区（OpenCode Web）

后端通过 `exec.Command("opencode", "serve", ...)` 管理服务进程，前端通过 Wails 绑定方法代理 API 请求。

### 服务管理

| 功能 | 后端方法 | 说明 |
|------|---------|------|
| 启动服务 | `StartOpenCodeWeb(port, hostname, proxy)` | 启动 opencode serve，等待端口就绪后返回状态 |
| 停止服务 | `StopOpenCodeWeb()` | 停止 serve 进程及子进程，同时关闭 SSE 事件流 |
| 状态检测 | `GetWebStatus(hostname, port)` | 通过 `/global/health` 检测运行状态、版本号 |
| 启动检测 | `domReady()` → `app-ready` 事件 | DOM 就绪后自动检测已运行的服务 |
| 终端打开 | `LaunchWindowsTerminal(mode, url, dir)` | 在 Windows Terminal 中打开指定目录 |

### 服务生命周期

```
应用启动 → domReady 发射 app-ready 事件
         → 前端 checkWebStatus(config.host, config.port)
         → 若已运行：加载项目树、启动事件流
         → 若未运行：显示"未启动"状态

点击启动 → StartOpenCodeWeb(port, hostname, proxy)
         → 注入 HTTP_PROXY/HTTPS_PROXY 环境变量
         → 等待 TCP 端口就绪（最多 12s）
         → 返回健康状态和版本

应用关闭 → shutdown() → StopOpenCodeEvents() + StopOpenCodeWeb()
```

### 网络配置

通过弹窗配置服务地址/端口和 HTTP 代理，持久化到 `localStorage('oc-manager-proxy-config')`。

| 字段 | 默认值 | 说明 |
|------|--------|------|
| serviceHost | `127.0.0.1` | OpenCode 服务地址 |
| servicePort | `4096` | OpenCode 服务端口 |
| proxyEnabled | `false` | 是否启用 HTTP 代理 |
| proxyHost | `127.0.0.1` | 代理地址 |
| proxyPort | `7897` | 代理端口 |

## 会话管理

### 项目树（左侧面板）

树形结构：**项目 → 工作目录 → 会话**。

| 操作 | 实现 |
|------|------|
| 加载树 | `GetProjectTree(knownDirs)` — 通过 `/project` 获取项目列表，`/session?all=true` 获取 git 项目会话，对 `knownDirs` 逐目录查询 global 会话 |
| 添加目录 | 项目标签旁 `＋` 按钮 → 选择目录 → 写入 `oc-known-dirs` → 刷新树 |
| 刷新树 | `buildTree()` → toast 提示成功/失败 |
| 展开/折叠 | `▼`/`▶` 箭头切换 `.oc-tree-children` 显隐 |

### 会话操作

| 操作 | 实现 | 说明 |
|------|------|------|
| 新建会话 | `createNewSession()` → 选择目录 → 设置 `pendingWorkDir` | 首次输入时才真正创建 |
| 创建会话 | `CreateSession(workDir)` → `POST /session` + header `x-opencode-directory` | body 为 `{}` |
| 命名会话 | 首次发送后 `PATCH /session/:id` 用输入前 15 字符作为标题 | |
| 删除会话 | `DELETE /session/:id` → 刷新树 | hover 显示 `✕` 按钮 |
| 切换会话 | `selectSession(id)` → `loadMessages()` → 强制滚到底部 | 右侧内容区同步更新 |

### 工作目录机制

- OpenCode 实例目录优先级：`directory` query → `x-opencode-directory` header → `process.cwd()`
- 服务启动目录默认为程序 exe 所在目录
- 会话目录通过 `x-opencode-directory` 请求头在 `POST /session` 时设置
- 已知目录持久化到 `localStorage('oc-known-dirs')`，用于 global 项目会话查询

## 消息区（中间面板）

### 消息渲染

| 消息类型 | 渲染方式 |
|---------|---------|
| 用户消息 | 右侧对齐，蓝色边框 |
| 助手消息 | 左侧对齐，灰色卡片 |
| 工具调用 | 可折叠卡片，按类型分类（shell/file/tool），显示输入/输出/状态 |
| 推理过程 | 折叠的 reasoning 块 |
| 步骤标记 | step-start / step-finish 分割线 |
| Markdown | `marked.parse()` 渲染 |
| 内部消息 | 系统提示等过滤不显示 |

### 性能优化

| 优化项 | 实现 |
|--------|------|
| 事件批处理 | `scheduleRenderCachedMessages()` 用 `requestAnimationFrame` 合并高频事件 |
| 增量更新 | 流式输出时仅追加新增片段，不全量重建 DOM |
| 消息缓存 | `messageCache[sessionID]` 按会话隔离缓存，事件驱动增量更新 |
| 滚动守卫 | `userScrolling` 标志跟踪鼠标拖拽，拖拽滚动条时跳过 DOM 重建 |
| CSS 隔离 | `.oc-messages` 使用 `contain: paint style`（不含 layout） |

### 滚动行为

| 场景 | 行为 |
|------|------|
| 切换会话 | `smartScroll(box, true)` → 强制滚到底部 |
| 发送消息 | 乐观渲染 → 滚到底部 → 加载正式响应 → 再滚到底部 |
| 流式输出 | `restoreScroll` 根据 `nearBottom`（120px 阈值）决定是否跟随 |
| 点击到底 | `scrollMessagesToBottom()` → `lastElementChild.scrollIntoView({ block:'end', behavior:'smooth' })` |
| 拖拽滚动条 | `userScrolling` 守卫阻止 DOM 重建打断操作 |

## 右侧面板

三个可折叠区块：

### 服务状态

显示 OpenCode 服务的 URL、健康状态（在线/离线/异常/未知）和版本号。

通过 `GetWebStatus()` 获取 → `/global/health` 解析 JSON 中的 `version` 字段。

### 代办事项

从当前会话的 assistant 消息中提取最后一次 `todowrite` 工具调用的 todos 数组。

| 显示 | 逻辑 |
|------|------|
| 进行中 | 按优先级色条（高=红、中=黄、低=灰）分组 |
| 已完成 | 降低透明度 + 删除线 |
| 全部完成 | 显示"会话中暂无代办" |
| 更新时机 | `renderMessages()` 末尾自动调用 `renderTodos()` |

### 修改

显示当前会话的文件变更 diff，通过 `GET /session/:id/diff` 获取，树形展示文件增减统计。

## 事件流

Go 后端通过 Wails 3 应用事件 `app.Event.Emit("oc-event", ...)` 向前端推送 OpenCode SSE 事件。

| 事件类型 | 前端处理 |
|---------|---------|
| `session.status` | 更新 busy/idle 状态，idle 时刷新消息和树 |
| `session.error` | 记录错误信息，当前会话则刷新消息 |
| `message.updated` | 更新缓存 → 调度渲染 |
| `message.part.updated` | 更新 part 缓存 → 调度渲染 |
| `message.part.delta` | 增量追加 part 内容 → 调度渲染 |
| `message.part.removed` | 移除 part → 调度渲染 |
| `message.removed` | 移除消息 → 调度渲染 |
| `session.created/deleted` | 重建项目树 |
| `session.updated` | 刷新 diff |

事件流通过 `StartOpenCodeEvents()` 启动，连接 `/global/event` SSE 端点。

|------|------|--------|

## 技术栈

| 层面 | 技术 |
