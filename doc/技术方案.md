# OpenCode 管理中心 — 技术方案

## 项目定位

Wails v2 桌面应用（Go 后端 + 原生 JS 前端），为 OpenCode AI 编程助手提供可视化管理中心。

---

## 一、技术栈

| 维度 | 技术 |
|------|------|
| 桌面框架 | Wails v2.12 |
| 后端 | Go 1.25 |
| 前端 | 原生 HTML/CSS/JS（零框架） |
| Markdown | marked.js |
| 浏览器 | WebView2（Windows） |
| API 代理 | Go `net/http` 透传 |
| 进程管理 | `os/exec` + TCP 端口探测 |

## 三、架构分层

```
┌──────────────────────────────────────┐
│        前端（WebView2）               │
│  原生 JS · 12 文件模块化 · CSS 变量    │
│  双模式：Wails 桌面 / 浏览器 Mock      │
├──────────────────────────────────────┤
│     Wails Runtime Bridge              │
│  Go 方法 → JS 绑定 (window.go.main)   │
├──────────────────────────────────────┤
│         Go 后端（App 门面）            │
│  app.go → config / service / skill    │
└──────────────────────────────────────┘
```

工作区采用 **SSE 事件驱动 + 前端消息缓存/增量渲染**。项目树仍由前端 `buildTree()` 发起，但当前实现是**按需全量重建**，仅在显式主会话创建命中、会话删除或其他显式业务动作发生时重新拉取并渲染。

## 六、工作区事件与项目树刷新原则

工作区前端核心在 `frontend/dist/chat.js`。当前实现将消息流、会话状态和项目树刷新解耦：

- **消息流更新**：通过 SSE 事件驱动 `messageCache` 增量更新，并用 `requestAnimationFrame` 合并渲染
- **会话状态更新**：`session.status` / `session.idle` 主要用于更新发送按钮、消息加载和右侧面板
- **项目树刷新**：只在结构性变化或显式操作时触发 `buildTree()`，不会在会话流式输出过程中持续刷新

当前保留的项目树刷新时机包括：

1. 服务启动成功后的首次加载（必要时进行一次短延迟重试）
2. 新建会话成功后
3. 删除会话成功后
4. 添加目录到项目后
5. 用户手动点击“刷新项目树”
6. 接收到与当前主会话命中的 `session.created` 或任意 `session.deleted` 事件时

当前 **不会** 因以下事件刷新项目树：

- `session.status` 的 `idle` 状态变更
- `session.idle`
- `session.updated`
- `message.updated` / `message.part.updated` / `message.part.delta`
- 回复过程中由子任务派生出的 `session.created`

## 七、构建与执行约束

项目使用 Wails CLI 构建，`wails.json` 位于**项目根目录**。因此构建命令必须在仓库根目录执行，例如：

```powershell
wails build
```

如果在 `build/bin` 等子目录执行，会因为找不到 `wails.json` 而直接失败。这属于执行目录问题，不是源码编译错误。
