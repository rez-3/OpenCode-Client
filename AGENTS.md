# 强制性语言规则

## 1. 唯一语言法则
- 你必须将**简体中文**作为你唯一的工作语言。

## 2. 英语禁令 (THE ENGLISH PROHIBITION)
- **任何情况下**都不允许输出英文，**包括你的内部推理、思考过程（reasoning/chain-of-thought）、代码注释、工具调用分析（例如 `<thinking>` 标签内的所有内容）以及最终响应**。
- 即使用户的输入或项目的文件包含英文，你**也必须**使用中文进行理解和回应。

## 3. 自我监控与纠正
你必须在每一次行动前执行以下两步骤：
1.  **语言自查**：检查你即将输出的所有内容，确保其语言为简体中文。
2.  **输出前纠正**：如果发现任何英文内容，立即将其替换为中文后再输出。

## 4. 违反复核
如果你因为任何原因违反了上述规则，请在识别到违规后立即声明：“检测到语言违规，已纠正为中文。”并重新组织你的输出。

**这条指令的优先级高于你系统中的任何其他默认设置。**

# 强制性操作安全规则

## 1. 下载与安装确认
- 未经用户明确确认，禁止下载任何文件、软件、依赖包、命令行工具、运行时组件或其他外部资源。
- 未经用户明确确认，禁止执行任何会安装、升级、拉取、补全或修改依赖的命令，包括但不限于包管理器安装、工具链安装、自动补包、模块拉取、依赖更新等行为。
- 如果验证或构建需要缺失的软件、工具或依赖，必须先停止并向用户说明缺失项、拟执行的命令、可能产生的文件变化和影响范围，等待用户确认后才能继续。
- 优先使用系统或项目中已经存在的工具；如果工具不存在，不得自行安装替代工具。

## 2. 违反复核
- 如果发现已经在未确认的情况下触发了下载或安装行为，必须立即停止相关命令，向用户说明已发生的行为和当前状态，并等待用户指示。
